# Codex UI 对齐矩阵（脱敏基准）

> 采样日期：2026-07-22。基准来自 Codex 桌面的只读 Computer Use 截图与可访问树；本文只记录结构、状态和 Grok 功能映射，不保存账号、任务标题、会话正文、绝对路径或截图。

## 0.6.4 修正：右侧是按需工作区，Review 是其中一个真实工具

后续 Codex 实际截图表明，右侧区域可以承载审核、计划/结果文档、文件和侧边任务，而不是永久 Review 或自造会话摘要。0.6.4 使用一个通用开关和启动器映射现有后端：

| 右侧工具 | Grok 映射 | 行为与边界 |
|---|---|---|
| 审阅 | Unstaged、Staged、Commit、Branch、Last turn | 先加载轻量文件索引，一次只取选中文件 Patch；非 Git 为普通空状态 |
| 计划/结果 | 当前会话最近真实计划或最终回答 | 复制、在主会话展开、关闭；不增加点赞/语音/反馈 |
| 最近文件 | 最近回合实际写入路径及只读预览 | 显式选择“查看 Diff”或“编辑文件”后才切换中央工作台 |
| 侧边任务 | Agent、后台任务、队列、等待和会话状态 | 只读映射已有任务数据；没有假终端/浏览器入口 |

- 面板宽度为 `clamp(420px, 42vw, 760px)` 并按工具保存；窄窗使用覆盖抽屉，不再由媒体规则强制隐藏。
- File 和 Review 不再占据左侧开发工具常驻项。左侧保持项目/任务优先，低频能力在默认折叠的“开发工具”内。
- Dashboard、文件、任务中心和会话入口共用导航控制；中央工作台始终有“返回会话”，返回后重新测量消息区并恢复输入框。
- Review 的 850 文件场景不再把全部 Patch/hunk 一次注入 Renderer DOM；右侧文件树搜索并选择单文件，主区只呈现该文件 unified diff。

## 0.6.4 提供商界面映射

提供商不是 Codex 右栏的伪工具，而是独立管理弹窗。设置与账号页只提供统一入口；管理器使用列表＋详情/向导，草稿保存前即可测试模型列表、搜索/多选导入或手工添加。探测不发送推理请求，未知上下文保持未知。

## 0.6.2 审计修正：右栏是 Review，不是通用会话摘要

0.6.1 将自造的“会话摘要栏”误写为 Codex 对齐。0.6.2 删除该映射：Codex 右侧面板的核心是代码审核，任务信息回到任务标题菜单，Review 默认关闭且只在用户点击标题按钮、文件修改卡或审核入口时出现。

| Codex 状态/操作 | 0.6.2 Grok 映射 | 真实后端与边界 |
|---|---|---|
| Unstaged | 工作树相对索引的 unified diff | `git diff`；未跟踪文件以 `git diff --no-index` 读取 |
| Staged | 索引相对 HEAD 的 unified diff | `git diff --cached` |
| Commit | 选定提交的只读 Patch | 解析校验后的 revision；不提供写操作 |
| Branch | merge-base 到 HEAD 的只读 Patch | 分支名由 Git 校验；不提供写操作 |
| Last turn | 当前 Git 变更与最近回合真实写入路径的交集 | 路径来自工具写入位置并以会话/Worktree `cwd` 为根 |
| 文件状态与统计 | A/M/D/R/C/U/冲突、二进制、增删行 | 主进程解析 Patch 和 porcelain-v2；Renderer 不提交任意 Patch |
| 文件级暂存/取消暂存/恢复 | Review 文件操作 | 固定 Git 参数数组；恢复要求明确确认 |
| 区块级暂存/取消暂存/恢复 | hunk 标题栏操作 | 快照 ID、文件 ID、hunk ID 重新校验；Patch 仅来自主进程快照并通过 stdin |
| 行级批注 | Diff 行悬停 `+`，作为输入框上方草稿 | 发送时转换成可见 `@path#Lx` 引用；快照过期阻止发送并保留草稿 |
| 打开完整 Diff | 切换中央 Git 工作台并选择对应文件 | 使用同一 `NavigationIntent` 与执行根目录 |
| 编辑文件 | 显式进入 Monaco 编辑模式 | 默认只读；编码、换行、原子保存和外部冲突保持原实现 |

## 过程与结果状态

- 新回合：运行时展开为“正在处理 · 1:23”；完成、失败、取消均持久化结果和真实耗时并自动折叠。
- 旧回合：没有原始计时数据时仅显示“已处理”；连续无用户消息/最终回答的过程段合并为“历史执行记录（N 段）”。
- 同一工具 ID 的增量更新仍由 reducer 合并；隐藏思考时每组只保留一条汇总。
- 每回合文件写入显示一个“修改了 N 个文件”组，并可直接打开 Last turn Review。
- 最终回答位于过程块外；复制和真实分叉可用。图片/视频位于最终结果区，可大图或打开原文件。
- 背景不再叠加固定主色覆盖；`opacity=100%`、`dim=0`、`blur=0` 时空白画布显示原图，文字可读性由局部表面承担。

## 设计基线

| 区域 | Codex 观察 | Grok 0.6.1 映射 | 验收重点 |
|---|---|---|---|
| 应用框架 | 单一安静侧栏、约 48 px 标题栏、约 740 px 会话列、浮动输入框 | 270 px 左栏、52 px 标题栏、760 px 会话列、320 px 可折叠摘要栏 | 1280×720 到 1920×1080 无重叠；窄窗口可折叠 |
| 左侧全局导航 | 产品切换、搜索、新任务、少量一级功能 | Grok、搜索、新建任务、文件、源代码管理、Worktree、任务、扩展 | 不再使用九宫格或工作台下拉；入口有图标和文字 |
| 项目与任务 | 项目分组、项目操作、新任务、任务列表、悬停置顶/归档 | 工作区分组、会话来源分组、悬停置顶/归档/更多 | 当前项、运行态、未读态清晰；操作不遮住标题 |
| 项目工具 | 非高频能力不挤占主导航 | Memory、Agent/Persona、Profiles、Dashboard 放入可折叠项目工具 | 默认收起但可发现；保留真实能力 |
| 标题栏 | 任务菜单、打开位置、摘要/面板/侧栏开关 | 同结构映射到会话历史、资源管理器、摘要栏、环境面板和侧栏 | 所有按钮有 tooltip、aria-label 和明确选中态 |
| 会话主体 | 用户气泡、处理时长、折叠过程、修改文件卡、最终回答 | 用户消息、执行过程、工具分组、变更摘要、最终回复 | 完成过程默认折叠；运行过程展开；正文不被背景吞没 |
| 消息动作 | 用户复制；回复复制、继续任务等上下文动作 | 用户复制；回复复制；能力支持时分叉；文件卡跳转 Editor/Git | 不显示没有后端的点赞、语音或 PR 占位按钮 |
| 输入框 | 添加内容、权限、模式、模型、发送/停止；运行时状态切换 | 附件/能力、权限、Agent/Plan/Auto、模型/强度、排队/插话、发送/停止 | 空闲、聚焦、附件、运行、等待用户五类状态可辨 |
| 右侧摘要 | 按需显示任务上下文与摘要 | 计划、修改文件、状态、模型、上下文、Memory、Worktree、等待事项 | 宽屏可打开；小于 1200 px 自动收起且保留开关 |
| 底部环境栏 | 运行位置、变更、分支、提交/推送 | 本地工作区、Git 变更、分支、Worktree、Commit/Push | 不伪造 PR；网络动作使用现有取消和错误处理 |
| 附件 | 发送前可见，发送后属于用户消息 | 图片缩略图/网格、文件卡、大图查看、发送/失败状态 | 纯图片、多图、队列、重开会话和源文件缺失均有表示 |
| 主题与背景 | 中性深色、高对比度、弱边框层级 | 保留用户自定义背景，但增加稳定遮罩和消息表面 | 文本、代码、按钮和缩略图在背景上仍满足可读性 |

## 状态验收清单

- 左栏：展开、折叠、搜索聚焦、项目折叠、会话悬停、会话菜单、账号与设置。
- 标题栏：无会话、活动会话、Codex 镜像、忙碌、摘要栏开关、环境面板开关。
- 会话：用户文本、用户图片、思考、命令、文件修改、Computer Use、权限、问题、计划、错误、最终回复。
- 输入框：空白、文字、单图、多图、能力 chip、运行中排队、Ctrl+Enter 插话、停止和发送失败恢复。
- 响应式：1280×720、1440×810、1920×1080、960 px 以下和系统文字缩放组合。

## 固定边界

- 只映射 Grok 已实现能力；不创建不可用的站点、PR、语音或反馈按钮。
- Renderer 继续沙箱化；磁盘、缓存、Git、ACP 和凭据处理均留在主进程。
- UI 验收夹具不得发送真实提示词或使用付费模型。
